## 5. 进阶内容观察笔记

### 笔记 1：AI 时代 Web3 开发者需要什么能力

**关注对象**：TC（@tc996271334）  
**讨论的问题**：AI 工具正在改变开发者的工作方式，Web3 开发者不只需要会写合约或前端，也需要理解系统架构、安全边界、链上数据和产品验证。  
**技术 / 产品趋势**：AI coding 会降低开发门槛，但不会替代问题定义、架构判断和安全意识。  
**与 Week 1 的关系**：Week 1 的目标是建立基础工具和信息流，这说明新手需要先搭好学习系统，而不是直接追复杂技术。  
**还不理解的问题**：AI coding 生成的智能合约或链上交互代码，应该如何系统性审计和测试？

### 笔记 2：AI Agent 从 0 到 1

**关注对象**：Draken（@draken_zeng）  
**讨论的问题**：Agent 不只是聊天机器人，而是能使用工具、保持上下文、规划任务并执行动作的系统。  
**技术 / 产品趋势**：未来 AI × Web3 产品可能从“AI 解释链上数据”走向“AI 代表用户执行链上任务”。  
**与 Week 1 的关系**：Week 1 需要理解 AI 工具的基本能力和边界，为后续选择 Agent / API / 钱包工具打基础。  
**还不理解的问题**：Agent 如果能自动发起交易，哪些步骤必须保留人工确认？

### 笔记 3：Web3 运行原理与 Ethereum 基础

**关注对象**：Bruce Xu（@brucexu_eth）  
**讨论的问题**：Web3 的核心不是“发币”，而是账户、交易、合约、状态、验证和开放协作网络。  
**技术 / 产品趋势**：真正可持续的 Web3 项目需要理解底层运行机制，而不是只做一个接钱包的页面。  
**与 Week 1 的关系**：这和 Week 1 的钱包、区块浏览器、测试网、Remix 等基础工具准备直接相关。  
**还不理解的问题**：一个普通用户如何判断某个 Web3 产品是否真的需要上链？

### 笔记 4：Open Agentic Economy 与 Ethereum Builder Path

**关注对象**：Ethereum（@ethereum）、Austin Griffith（@austingriffith）  
**讨论的问题**：当 AI Agent 可以代表用户发现机会、调用工具、执行交易时，需要新的身份、权限、支付、声誉和安全基础设施。  
**技术 / 产品趋势**：Agent 经济可能需要账户抽象、intent、可验证执行、权限控制和长期记忆等能力共同支持。  
**与 Week 1 的关系**：Week 1 先建立信息流，可以帮助我持续观察 Agent、钱包、安全、Ethereum 生态之间如何连接。  
**还不理解的问题**：Agent 之间的身份、信誉和责任边界应该由链上协议、钱包产品还是应用层来定义？
